# Connect Mobile — документация для фронтенд-разработчика

Гайд по коду дизайн-системы: что за что отвечает, какие пропсы, как выглядит, где используется. Всё построено на CSS-переменных (токенах) и React-компонентах без сторонних UI-библиотек. Иконки — [Lucide](https://lucide.dev) через CDN.

Источник правды по логике страниц Заказы/Финансы/Чат — репозиторий [Tvinck/connect-mobille](https://github.com/Tvinck/connect-mobille).

---

## 1. Структура проекта

```
styles.css                     — точка входа: только @import
tokens/
  colors.css                   — цвета + светлая тема [data-theme="light"]
  typography.css               — шрифт, размеры, начертания
  spacing.css                  — отступы, радиусы, высоты
  base.css                     — сброс, body, ссылки
components/
  core/     Button, Badge, Switch, Field, Icon
  layout/   Card, NavBar, TabBar
  content/  IconTile, ListRow, ServiceTile, StatCard,
            PromoBanner, NotificationItem, ChatBubble, ProjectPicker
ui_kits/connect-mobile/        — экраны приложения (см. раздел 4)
guidelines/*.card.html          — карточки-образцы для вкладки Design System
```

Компилятор сам собирает всё в `window.ConnectMobileDesignSystem_60c434`. В HTML подключаешь `_ds_bundle.js` и берёшь компоненты из этого объекта. **Не** подключай `.jsx` напрямую.

## 2. Токены (CSS-переменные)

Все значения — на `:root` (тёмная тема по умолчанию). Светлая тема — переопределение тех же переменных внутри `[data-theme="light"]`. Чтобы включить светлую тему, повесь `data-theme="light"` на любой контейнер-предок (в приложении это `.phone`).

### Цвета — `tokens/colors.css`
| Переменная | Тёмная | Назначение |
|---|---|---|
| `--bg` | `#000000` | фон приложения |
| `--surface` | `#1c1c1e` | карточка |
| `--surface-2` | `#2c2c2e` | вложенный элемент (кнопка/плитка в карточке) |
| `--surface-3` | `#3a3a3c` | нажатое/третичное заполнение, иконочные плитки |
| `--hair` / `--hair-strong` | rgba белого 6% / 12% | разделители, границы |
| `--text` / `--text-2` / `--text-3` | white / `#a1a1a6` / `#6e6e73` | 3 уровня текста |
| `--accent` | `#0a84ff` | синий: ссылки, активный таб, колокольчик, текст-кнопки |
| `--yellow` / `--on-yellow` | `#ffdd2d` / `#1c1c1e` | главная CTA-кнопка и текст на ней |
| `--green/red/amber/violet` (+ `-dim`) | — | статусы; `-dim` = 15% подложка |
| `--promo-grad` | зелёный градиент | промо-баннер, welcome-иконка |
| `--tabbar-bg` | rgba тёмного 92% | фон нижнего меню (в светлой — белый 90%) |

Семантические алиасы: `--surface-card`, `--text-body`, `--link`, `--nav-active`, `--nav-inactive`. Используй их в новом коде — при смене темы всё подхватится.

### Типографика — `tokens/typography.css`
`--font-sans` — системный стек (SF Pro на iOS). Размеры iOS-шкалы:
`--fs-large-title 34` (заголовок таба) · `--fs-title 22` (заголовок секции) · `--fs-headline 17` (нав-бар, кнопки, строки) · `--fs-subhead 15` (подписи, чат) · `--fs-footnote 13` (время, мета) · `--fs-caption 11` (подписи табов). Начертания: `--fw-regular/medium/semibold/bold`.

### Отступы и радиусы — `tokens/spacing.css`
Сетка 4px: `--sp-1…--sp-8` (4→32). Радиусы: `--radius-sm 10` (бейджи) · `--radius 16` (кнопки/поля/плитки) · `--radius-lg 24` (карточки/баннеры) · `--radius-pill` (чипы, тумблеры). Плюс `--screen-pad 16`, `--card-pad 20`, `--tabbar-h 82`.

## 3. Компоненты

### core/

**Button** — `variant` (`primary` жёлтая CTA / `tonal` тёмная с синим текстом / `plain` синий текст / `blue` синяя заливка), `icon`, `block`, `disabled`, `onClick`. Нажатие: `scale(0.98)` + прозрачность 0.75, 150 мс. В карточках всегда `block`. Одна `primary` на экран.

**Badge** — маленький статус-ярлык. `tone` (`neutral/blue/green/red/amber/violet`) — подложка `-dim` + цветной текст. Радиус `--radius-sm`.

**Switch** — iOS-тумблер 51×31, синий во включённом. Управляемый (`checked`) или неуправляемый (`defaultChecked`), `onChange`.

**Field** — тёмное скруглённое поле ввода. `value/onChange/placeholder/type`. Используется в чат-композере и форме входа.

**Icon** — обёртка над Lucide. `name` (kebab-case, напр. `"message-square"`), `size`, `color`, `strokeWidth`, `fill`. Требует `<script src=".../lucide.min.js">` на странице. **Всё иконки только через него**, свои SVG не рисуем.

### layout/

**Card** — базовая поверхность: `--surface`, радиус 24, паддинг 20, без тени и границы. `inset` — светлее (`--surface-2`). `onClick` делает карточку кликабельной.

**NavBar** — верхняя панель. Два режима: `large` (крупный заголовок 34 слева + синий колокольчик справа, `bellDot` — красная точка) для корня таба; компактный (центр. заголовок 17 + `onBack` шеврон + `subtitle`) для вложенных экранов.

**TabBar** — нижнее меню. `items: [{key,label,icon,badge}]`, `active`, `onChange`. Фон `--tabbar-bg` + блюр, активный таб синий, красные счётчики. Позиционируется `absolute` — родитель должен быть `relative` (рамка телефона).

### content/

**IconTile** — скруглённый квадрат-«иконка приложения» с глифом Lucide. Ведущий визуал строк, плиток, уведомлений. `icon/color/bg/size/radius`.

**ListRow** — кликабельная строка: IconTile + заголовок/подзаголовок + шеврон. `icon/iconBg/iconColor/title/subtitle/onClick/right/chevron`. Основа списков сервисов и настроек.

**ServiceTile** — плитка для горизонтальной карусели «Полезные сервисы»: иконка сверху, подпись снизу. `icon/label/onClick`.

**StatCard** — карточка-метрика: иконка, крупное число, серая подпись. `value/label/icon/tone`. Счётчики на Главной, раскладывать в grid 2 колонки.

**PromoBanner** — зелёный градиентный баннер с большим глифом справа и кнопкой-крестиком. `title/subtitle/icon/onDismiss/onClick/gradient`. Один на экран.

**NotificationItem** — карточка уведомления/новости: круглая иконка, мета «категория • дата • время», жирный заголовок, серый текст. Цвет иконки кодирует категорию (жёлтый — новости, фиолетовый — расписание, синий — база знаний).

**ChatBubble** — пузырь мессенджера: `mine` (синий, справа) / чужой (серый, слева, с `author` и `avatar`), `time`, `read` (✓✓).

**ProjectPicker** — чип-переключатель проекта с зелёной точкой и выпадающим списком. `projects/value/onChange`. Над заголовком Главной (BAZZAR SERTS / Veil VPN).

## 4. Экраны приложения (`ui_kits/connect-mobile/`)

`index.html` — оболочка: рамка телефона 390×844, статус-бар, роутинг, состояние авторизации и темы. Всё состояние в `App()`.

### Оболочка и навигация
- **auth**: `'login' → 'welcome' → 'app'`. Выход возвращает в `'login'`.
- **theme**: `'dark' | 'light'` — кладётся в `data-theme` на `.phone`.
- **tab**: `home | hr | chat | more` (нижнее меню).
- **overlay**: вложенный экран поверх таба (`notifications`, `services`, `line`, `news`, `orders`, `finances`, `coming:<Заголовок>`). `null` = показываем таб. Пока есть overlay, нижнее меню скрыто.
- `open(key)` открывает overlay, `back()` закрывает.

### Экраны
| Файл | Что делает / как выглядит |
|---|---|
| **LoginScreen** | Вход. Логотип CONNECT, поля email + пароль, кнопка «Войти», ссылка «Не помню пароль». Валидирует, что поля непусты (макет — принимает любые данные). → `welcome`. |
| **WelcomeScreen** | Приветствие после входа: градиентная иконка, «Добро пожаловать, {имя}», кнопка «Войти в приложение». Имя берётся из части email до `@`. → `app`. |
| **HomeScreen** | Главная. ProjectPicker, 4 StatCard (сообщения/заказы/открытые/закрытые — зависят от проекта), карточки «Разметка данных» и «Выход на линию», PromoBanner, карусель «Полезные сервисы», блок «Справка и поддержка». Кнопки ведут в соответствующие overlay. |
| **ClientLineScreen** | Открывается по «Начать работу». Операторская линия: список клиентов, отсортирован по времени ожидания (зелёный <5мин / жёлтый 5–15 / красный ≥15), бейджи платформы и источника (Почта/Сайт/GGSel/Авито/Прочее), фильтр по проекту. Тап → переписка с полем ответа. |
| **ServicesHRScreen** | Таб «Сервисы HR». Список ListRow: Новости, TWFM, Статистика, Заказы, Финансы → open(ключ). |
| **UsefulServicesScreen** | То же содержимое, открывается по «Все» с Главной. |
| **NewsScreen** | Лента новостей (NotificationItem) по категориям BAZZAR SERTS / Veil VPN / Компания. |
| **OrdersScreen** | Заказы Apple Certs (по коду репозитория). Фильтры Новые/В работе/Все, карточка с UDID, тарифом, платформой, источником, ценой; кнопка циклит статус pending → in_progress → approved. |
| **FinancesScreen** | Финансы (по коду репозитория). Плитки Доходы/Расходы/Прибыль + список транзакций со знаком и цветом. |
| **NotificationsScreen** | Уведомления (колокольчик). Лента NotificationItem, сгруппирована «Ранее». |
| **MoreScreen** | «Ещё». Переключатель темы (сегмент Светлая/Тёмная → `onThemeChange`), «Очистить кеш», версия. Кнопка «Выйти» открывает шторку подтверждения (Выйти/Отмена) → `onLogout`. |
| **ComingSoonScreen** | Заглушка «Раздел в разработке». `title` — название раздела. Используется для TWFM, Статистика, Разметка данных, Идеи, Задачи, Информер. |

### Как добавить новый раздел
1. Создай `ui_kits/connect-mobile/MyScreen.jsx` с `export function MyScreen({ onBack }) {}`, собери из компонентов, начни с `<NavBar title="…" onBack={onBack} />`.
2. В `index.html`: добавь в деструктуризацию `DS`, добавь ветку в роутинг (`else if (overlay === 'my') …`).
3. Повесь `onClick={function(){ onOpen('my'); }}` на нужный ServiceTile/ListRow.

## 5. Что ещё НЕ реализовано (заглушки / неактивно)
- TWFM, Статистика, Разметка данных, Идеи, Задачи, Информер — ComingSoonScreen.
- Карточки уведомлений не кликаются (нет детального экрана).
- «Очистить кеш» — только меняет подпись; реальной очистки нет.
- Тема и позиция не сохраняются между перезагрузками (нет localStorage).
- Поиск в чате (лупа) и вложения (скрепка) — визуальные.
- Полный список HR-сервисов из фото (Букинг мест, События, Электронные документы, Магазин Т-Гейм и т.д.) намеренно сокращён до 5 по брифу.
- Восстановление пароля — ссылка без экрана.
- Данные везде замоканы в компонентах; сети/бэкенда нет.
