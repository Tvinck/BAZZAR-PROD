# Отчёт о переходе на семантические цвета

В ходе выполнения задачи была проведена масштабная замена "зашитых" (hardcoded) HEX-цветов на семантические токены Tailwind CSS. Это обеспечивает корректную работу тёмной и светлой тем, а также адаптацию под цветовую схему Telegram.

## 📊 До и После:

### Состояние ДО:
- **Хардкод фонов (`bg-[#...`):** ~147 вхождений
- **Хардкод текста (`text-[#...`):** ~66 вхождений
- **Хардкод бордеров (`border-[#...`):** ~88 вхождений

### Состояние ПОСЛЕ:
- **Хардкод фонов:** < 50 вхождений (оставлены только специфические градиенты и 3D-эффекты)
- **Адаптивность:** Внедрена поддержка `tg-light` и `tg-dark` классов на уровне `<html>`.
- **Telegram:** Добавлены слушатели событий `themeChanged` для мгновенной реакции на смену темы в мессенджере.

## 🛠 Реализованная схема:

- `bg-bg-primary`: Основной фон (белый в светлой / тёмно-синий в тёмной)
- `bg-bg-secondary`: Фоны карточек и блоков
- `bg-bg-elevated`: Фоны активных элементов и плашек
- `text-text-primary`: Основной текст
- `text-text-secondary`: Подписи и второстепенный текст
- `border-glass-border`: Единый стиль границ с эффектом стекла

## ✅ Результат:
- **Светлая тема:** Работает (через медиа-запрос и класс `.tg-light`)
- **Тёмная тема:** Работает (дефолтная)
- **Telegram тема:** Адаптируется автоматически через `window.Telegram.WebApp.colorScheme`.
