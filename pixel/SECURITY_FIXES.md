# 🛡️ SECURITY FIXES: BAZZAR-PIXEL-CLEAN

Дата: 2026-03-16
Статус: Исправлено

В ходе аудита были обнаружены и исправлены следующие критические уязвимости:

## 1. Защита админ-панели (Role-Based Access Control)
- **Файл:** `server/routes/admin.js`
- **Изменения:**
    - Добавлен мидлвар `requireAdmin`, который проверяет поле `role` в таблице `users` в Supabase.
    - Мидлвар применен ко **всем** эндпоинтам в `admin.js`.
- **Результат:** Теперь только пользователи с ролью `admin` могут обращаться к админ-функциям. Все остальные получат ошибку `403 Forbidden`.

## 2. Отключение отладочной авторизации (Allow Dev Auth)
- **Файл:** `server/middleware/auth.js`
- **Изменения:**
    - Полностью удалена возможность входа через `x-dev-auth-id`.
    - При попытке использования флага `ALLOW_DEV_AUTH` сервер теперь выбрасывает ошибку (`throw Error`), блокируя доступ.
- **Результат:** Исключен риск несанкционированного входа через отладочные механизмы в любой среде.

## 3. Изоляция пользовательских данных в /stats
- **Файл:** `server/routes/user.js`
- **Изменения:**
    - Эндпоинт `/api/user/stats` теперь защищен мидлваром `authTG`.
    - Удалено использование `botAnalytics` (Service Role Key).
    - Теперь запрос идет через стандартного клиента Supabase с принудительной фильтрацией по `telegram_id` текущего авторизованного пользователя.
- **Результат:** Пользователи могут видеть статистику **только** своего профиля. Утечка данных других пользователей через этот эндпоинт невозможна.
